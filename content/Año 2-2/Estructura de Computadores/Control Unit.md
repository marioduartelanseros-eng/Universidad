# Control Unit

- Component of the CPU that directs the operation of the datapath
- Interprets instructions
- Generates control signals
- Coordinates execution of operations

Works together with:
- [[Datapath]]
- [[ALU]]
- Register File
- [[Program Counter (PC)]]
- [[Tema 3 – Instruction Set Architecture (ISA)|Instruction Set Architecture]]

---

## Main Responsibilities

### 1. Instruction Decode
- Reads instruction from the Instruction Register
- Identifies:
	- Opcode
	- Source registers
	- Destination register
	- Immediate values

Related to:
- [[Tema 3 – Instruction Set Architecture (ISA)#Five Stages|Five Stages]]

---

### 2. Generate Control Signals

Control signals determine:

- ALU operation
- Register write enable
- Memory read / write
- Branch decision
- PC update source

Related to:
- [[Tema 3 – Instruction Set Architecture (ISA)#5. Memory Operands|Memory Operands]]

---

### 3. Control the Execution Flow

For each instruction:

1. Fetch
2. Decode
3. Execute
4. Memory access (if required)
5. Write back

This sequencing defines:
- [[Stored Program Computers]]
- [[Tema 3 – Instruction Set Architecture (ISA)#Five Stages|Five Stages]]

---

## Hardwired vs Microprogrammed Control

### Hardwired Control
- Control signals generated by combinational logic
- Faster
- Used in most RISC processors

### Microprogrammed Control
- Control signals generated by microinstructions stored in control memory
- More flexible
- More common in complex CISC designs

Related to:
- RISC vs CISC

---

## Example: ADD Instruction

For:
add x2, x10, x4

Control Unit:

- Selects ALU operation = ADD
- Enables register write
- Disables memory write
- Sets PC = PC + 4

The [[Datapath]] performs the actual addition.

---

## Why It Matters

- Without the Control Unit, the datapath would not know what to do
- It translates ISA-level instructions into hardware-level actions
- Directly affects:
	- CPU Time
	- Performance Summary

The Control Unit is the decision-making logic of the processor.
