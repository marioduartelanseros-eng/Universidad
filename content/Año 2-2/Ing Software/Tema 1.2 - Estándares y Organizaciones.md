# Necesidad de estandarizar
## Proceso tipo
Marco para valorar los procesos implantados e identificar puntos de mejora
## Conjunto mínimo de buenas practicas
Lo que funciona se vuelve habito en los procesos de desarrollo
## Lenguaje Común
mejora comunicación entre los diferentes roles

# Uso de Estándares
## Certificación
- Acceso a proyectos regulados
- Obtención de permiso para comercialización de productos y servicios

# Principales estándares
## ISO (Organización Internacional de la Estandarización)
- ISO 9126: estándar internacional para la evaluación de la calidad del software mediante métricas que analizan características básicas (funcionalidad, fiabilidad, usabilidad, eficiencia, mantenibilidad, portabilidad y satisfacción).
- Familia ISO 9000: gestión de la calidad
	- Norma ISO 9000: fundamentos y vocabulario
	- Norma ISO 9001: requisitos
	- Norma ISO 9004: directrices de mejora continua
- ISO/IEC 12207: estándar para los procesos del ciclo de vida de software. Agrupa las acciones que se llevan a cabo durante el ciclo de vida en procesos principales, procesos de apoyo y procesos organizativos.
- ISO/IEC 15504 (SPICE): estándar para la evaluación y determinación de la capacidad y mejora continua de procesos de ingeniería del software.

Define un conjunto de requerimientos para la evaluación del proceso del software. El objetivo del estándar es ayudar a las organizaciones a desarrollar una evaluación objetiva de cualquier proceso del software que produzcan.
# SPICE - Niveles de madurez

| Nivel         | Descripcion                                                                                                     |
| ------------- | --------------------------------------------------------------------------------------------------------------- |
| 0.Incompleto  | No hay implementación efectiva de procesos                                                                      |
| 1.Realizado   | Se implementan los procesos y se alcanzan los objetivos de los mismos.                                          |
| 2.Gestionado  | Se gestionan los procesos y se controlan los productos de trabajo de cada fase                                  |
| 3.Establecido | Los procesos que se siguen están basados en estándares.                                                         |
| 4.Predecible  | Se gestionan cuantitativamente los procesos, estableciendo objetivos.                                           |
| 5.Optimizado  | Mejora continua de los procesos, identificación de puntos débiles y búsqueda de buenas prácticas más efectivas. |
# CMMI
- Capability Maturity Model Integration
- Establecido por el SEI
- Actualmente gestionado por el CMMI institute
- Basado en SPICE y muy popular en USA
